# 프로젝트 구조 메모

이 문서는 `docs/project_context.md`를 보조하는 구조 설명 문서다. 기준 문서는 여전히 `docs/project_context.md`이며, 이 문서는 현재 저장소 골격이 그 기준을 어떻게 반영하는지 설명한다.

## 현재 저장소의 역할

현재 저장소는 질문력 Co-Learner 완성형 서비스를 구현하는 저장소가 아니다. 이 저장소의 현재 역할은 아래 두 가지다.

- 질문력 Co-Learner를 만들기 위한 AI Agent 팀 구조를 표현한다.
- 각 Agent가 구조화된 산출물을 생성하는 최소 순차 파이프라인 골격을 제공한다.

## 디렉토리별 역할

- `agents/`: 다섯 Agent와 순차 파이프라인 실행 골격
- `prompts/`: Agent별 프롬프트 초안 자리
- `schemas/`: Agent 입력/출력 스키마 초안
- `outputs/`: 실행 결과 저장 위치
- `examples/`: 실행용 예시 입력
- `docs/`: 기준 문서와 구조 설명 문서

## 구현 원칙

- `docs/project_context.md`를 single source of truth로 사용한다.
- 현재는 팀빌딩 단계이므로 외부 LLM API를 붙이지 않는다.
- 각 Agent는 구조화된 입력을 받아 구조화된 출력을 생성한다.
- 출력은 다음 Agent가 그대로 사용할 수 있는 JSON 직렬화 가능 형태를 우선한다.
- 기존 파일은 삭제하지 않고, 필요한 구조를 추가하는 방식으로 확장한다.

## 현재 포함된 최소 데모

`run_pipeline.py`는 예시 입력을 기반으로 5개 Agent를 순차 실행하고 `outputs/` 아래에 JSON 산출물을 기록한다. 이 데모의 목적은 모델 품질 검증이 아니라 팀 구조와 산출물 흐름을 확인하는 것이다.

## 다음에 이어질 작업 방향

- Agent별 프롬프트 초안 구체화
- 산출물 스키마 상세화
- 샘플 산출물 품질 개선
- 선택적 데모 레이어 또는 검증 로직 추가
